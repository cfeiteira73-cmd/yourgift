# YourGift OS — Deployment Guide

> Production-ready deployment on Render (API + Web) with Cloudflare edge security.

---

## Architecture

```
Internet → Cloudflare Edge Worker (WAF + rate limit)
              ↓
        Render (Frankfurt)
         ├── yourgift-api  (NestJS 10, port 3001)
         └── yourgift-web  (Next.js 14, port 3000)
              ↓
        Supabase (PostgreSQL, Frankfurt)
        Upstash Redis (BullMQ queues)
        AWS S3 + CloudFront (file storage)
```

---

## 1. First Deploy (Render Blueprint)

1. Push this repo to GitHub
2. Go to [Render Dashboard](https://dashboard.render.com) → **New → Blueprint**
3. Connect your GitHub repo — Render will read `render.yaml` automatically
4. Render creates `yourgift-api` and `yourgift-web` services
5. Fill in all `sync: false` env vars (see section below)
6. Render triggers the first deploy

### Required Secrets (set in Render Dashboard)

| Variable | Description | Where to get it |
|---|---|---|
| `DATABASE_URL` | Supabase pooler connection | Supabase → Project → Database → Connection pooling |
| `DIRECT_URL` | Direct Supabase connection (Prisma migrations) | Supabase → Project → Database → Direct connection |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL | Supabase → Project → Settings → API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon key | Supabase → Project → Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role | Supabase → Project → Settings → API |
| `GOOGLE_CLIENT_ID` | Google OAuth client | Google Cloud → Credentials → OAuth 2.0 |
| `GOOGLE_CLIENT_SECRET` | Google OAuth secret | Google Cloud → Credentials → OAuth 2.0 |
| `UPSTASH_REDIS_URL` | Redis for BullMQ | Upstash → Database → REST URL |
| `STRIPE_KEY` | Stripe secret key | Stripe Dashboard → Developers → API Keys |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook signing secret | Stripe → Webhooks → yourgift endpoint → Signing secret |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | Stripe publishable key | Stripe Dashboard → Developers → API Keys |
| `MIDOCEAN_KEY` | Midocean API key | Midocean → Developer Portal |
| `RESEND_API_KEY` | Resend email API key | resend.com → API Keys → New |
| `AWS_ACCESS_KEY_ID` | AWS IAM key for S3 | AWS IAM → Users → Security credentials |
| `AWS_SECRET_ACCESS_KEY` | AWS IAM secret | AWS IAM → Users → Security credentials |
| `SENTRY_DSN` | Sentry error tracking | sentry.io → Project → Settings → Client Keys |
| `BETTERSTACK_HEARTBEAT_URL` | Uptime monitoring | betterstack.com → Monitors → New Heartbeat |
| `BETTERSTACK_SOURCE_TOKEN` | Log shipping | betterstack.com → Logs → Sources |
| `NEXT_PUBLIC_BETTERSTACK_KEY` | RUM (browser monitoring) | betterstack.com → Real User Monitoring |

**Note**: `JWT_SECRET` is auto-generated by Render — do not set it manually.

## 2. Database Setup (Supabase)

```bash
# Run Prisma migrations against Supabase
DATABASE_URL="direct connection string" \
DIRECT_URL="direct connection string" \
pnpm --filter api exec prisma migrate deploy

# Verify schema
pnpm --filter api exec prisma studio
```

Enable **Point-in-Time Recovery (PITR)** in Supabase Dashboard → Settings → Addons (requires Pro plan).

## 3. Stripe Webhook

1. Stripe Dashboard → Developers → Webhooks → **Add endpoint**
2. URL: `https://yourgift-api.onrender.com/api/v1/payments/webhook`
3. Events to listen:
   - `payment_intent.succeeded`
   - `payment_intent.payment_failed`
   - `charge.dispute.created`
   - `invoice.paid`
   - `customer.subscription.deleted`
4. Copy **Signing secret** → set as `STRIPE_WEBHOOK_SECRET` in Render

## 4. Cloudflare Edge Worker

```bash
cd infra/cloudflare

# Install wrangler
npm install -g wrangler

# Login to Cloudflare
wrangler login

# Deploy to production
wrangler deploy --env production
```

The worker routes to `yourgift-api.onrender.com/*` and `www.yourgift.pt/*`.
See `infra/cloudflare/worker.ts` for the WAF logic.

## 5. GitHub Actions CI/CD

Add these secrets to **GitHub → Settings → Secrets and variables → Actions**:

| Secret | Value |
|---|---|
| `RENDER_API_KEY` | Render Dashboard → Account → API Keys |
| `RENDER_API_SERVICE_ID` | Render → yourgift-api → Settings → Service ID (`srv-…`) |
| `RENDER_WEB_SERVICE_ID` | Render → yourgift-web → Settings → Service ID (`srv-…`) |
| `CLOUDFLARE_API_TOKEN` | Cloudflare → My Profile → API Tokens → Create Token |
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare → Overview → Account ID |

Add this **variable** (not secret) to enable automatic worker deploy:

| Variable | Value |
|---|---|
| `CLOUDFLARE_WORKER_ENABLED` | `true` |

After configuration, every push to `master`:
1. TypeScript check (all packages)
2. Prisma generate
3. API build smoke test
4. Trigger Render deploy for API + Web in parallel
5. Optionally deploy Cloudflare Worker

## 6. Enterprise SSO (per tenant)

### OIDC (Okta, Azure AD, Google Workspace)

1. Admin panel → **SSO / SCIM** → **Add SSO Config**
2. Protocol: OIDC
3. Issuer URL: `https://your-org.okta.com` (or Azure tenant URL)
4. Client ID + Secret: from your IdP application config
5. Callback URL: `https://yourgift-api.onrender.com/enterprise-identity/oidc/:tenantId/callback`
6. Email domains: `acme.com, acme.co.uk`

### SAML 2.0 (Okta, Azure AD, ADFS, PingFederate, OneLogin)

Native implementation — no additional packages required.

1. Admin panel → **SSO / SCIM** → **Add SSO Config**
2. Protocol: SAML
3. **IdP Entry Point URL**: from IdP app config (e.g. Okta → Sign On → Identity Provider SSO URL)
4. **SP Issuer / Entity ID**: `https://api.yourgift.pt/enterprise-identity/saml/<tenantId>`
5. **IdP Signing Certificate**: paste the PEM from the IdP (Okta → Sign On → Signing Certificates → Download)
6. **ACS URL** (set in IdP app): `https://api.yourgift.pt/enterprise-identity/saml/<tenantId>/callback`
7. **SP Metadata** (set in IdP app if required):
   - Entity ID: same as SP Issuer above
   - NameID format: `urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress`
   - Assertion signature algorithm: `RSA-SHA256`
8. Email domains: `acme.com, acme.co.uk`

### SCIM (automated user provisioning)

1. Set env var: `SCIM_TOKEN_<TENANT_ID>=<bearer_token>` in Render
2. In Okta/Azure AD SCIM app:
   - Base URL: `https://yourgift-api.onrender.com/scim/v2/tenants/<tenantId>`
   - Bearer token: the value you set above
3. ServiceProviderConfig: `https://yourgift-api.onrender.com/scim/v2/ServiceProviderConfig`

## 7. Custom Domain

1. Render → yourgift-web → Settings → **Custom Domains** → `www.yourgift.pt`
2. Render → yourgift-api → Settings → **Custom Domains** → `api.yourgift.pt`
3. Add CNAME records in Cloudflare DNS pointing to Render hostnames
4. Cloudflare proxy: **orange cloud (proxied)** to enable WAF + CDN

## 10. Rollback

Render keeps the last 5 deploys. To rollback:
1. Render Dashboard → yourgift-api → **Deploys**
2. Click any previous deploy → **Rollback to this deploy**

For database rollback:
```bash
# List migrations
pnpm --filter api exec prisma migrate status

# Rollback last migration (destructive — take a backup first)
pnpm --filter api exec prisma migrate resolve --rolled-back <migration_name>
```
